## Key Decision Points

1. **Include pre-computed results?** If yes, users can verify figures/tables without re-running. If no, repo is smaller and forces reproduction. **Recommendation**: Include them in a `results/precomputed/` directory alongside the empty `results/metrics/` for fresh runs.

2. **GPU optional?** Yes — `torch_ops.py` already has CPU fallback. Make `torch` an optional dependency (`[gpu]` extra).

3. **Pseudonymise actor labels?** The paper says "firm identities are replaced with opaque labels." Check if the registry already uses opaque IDs or if this needs a scrubbing step.

4. **Include the full interfaces.py?** It has ~60 dataclasses for the full SMIM pipeline. The scripts only use `ModalFrame` and `DecompositionMethod`. Strip to minimal needed types to avoid confusion.
